## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, progress bar height (2px exception) |
| md | 16px | Default element spacing, notification internal padding (20px = 5 Tailwind units) |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Notification card uses `p-5` (20px) — acceptable as 4px-grid multiple
- Progress bar height: 2px — visual element, sub-grid exception allowed
- Icon container: `w-10 h-10` (40px) — exact 8pt multiple, touch-safe
- Notification card width: min 320px, max 400px — content-driven range
- Notification stack top offset: `top-24` (96px) — clears mobile nav bar

Source: `SystemNotification.tsx` — existing established values, carry forward unchanged.

## Interaction Contract

### Queue Behavior
- Maximum 3 notifications visible simultaneously at `top-24 left-1/2 -translate-x-1/2`
- Implementation: `notifications.slice(0, 3)` in render — no separate queue array in state
- When a visible notification dismisses, the next queued notification slides in from FIFO order
- URGENT notifications occupy a normal queue slot (no priority override)

### Dismiss Interactions
- **Click notification body:** dispatches `DISMISS_NOTIFICATION(id)` immediately
- **X button (URGENT only):** dispatches `DISMISS_NOTIFICATION(id)` immediately
- **Auto-dismiss timer:** fires after per-type duration, dispatches `DISMISS_NOTIFICATION(id)`
- **No "clear all" button** — 3-visible cap makes individual dismiss sufficient

### Enter / Exit Animation (existing Framer Motion — carry forward)
- Enter: `opacity: 0→1, y: -20→0, scale: 0.9→1, blur: 10px→0` over default spring
- Exit: `opacity: 1→0, scale: 1→1.05, blur: 0→10px` over `0.2s`
- AnimatePresence wraps the list — exit animation fires automatically when item removed from array

### Progress Bar
- `motion.div` at card bottom: `scaleX: 1→0`, `transformOrigin: left`
- Duration **must match** per-type auto-dismiss timeout (currently hardcoded to `6` — fix to type-based value)
- Height: 2px, `opacity-40`, color: type-driven accent hex
- URGENT notifications: no progress bar rendered (no auto-dismiss)

### Scanning Line Effect (existing — carry forward)
- `motion.div` inside card, `y: 0%→100%→0%`, 3s loop, `opacity-10`, `pointer-events-none`

## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label padding |
| sm | 8px | Compact element spacing, badge padding |
| md | 16px | Default element spacing, modal inner padding |
| lg | 24px | Section padding within modal |
| xl | 32px | Modal body padding, major section gaps |
| 2xl | 48px | Vertical rhythm between content blocks |

Exceptions:
- Guide button on exercise card: minimum touch target 44px height (mobile workout context)

- Modal full-screen overlay: `inset-0` with `p-4` (16px) outer padding on mobile

## Interaction Contracts

### Guide button ("?") on exercise card
- Visible at all times during select phase — not gated by exercise selection
- Tap opens ExerciseGuideModal with that exercise's data
- Timer/reps activity behind modal continues (modal is non-blocking to game state)
- `AnimatePresence` + `motion.div` with `initial={{ opacity: 0, scale: 0.95 }}` / `animate={{ opacity: 1, scale: 1 }}` / `exit={{ opacity: 0, scale: 0.95 }}`

### Text guide generation (first visit)
- `GET /api/exercise-guide?exerciseId={id}` fires on modal open
- If DB hit: guide JSON renders immediately
- If DB miss (first visit): skeleton pulse renders, Ollama generates, DB saves, content renders
- No retry button — silent fallback (empty sections hidden, not shown as error) per Phase 13 pattern

### Visual unlock button
- Hover: neon cyan glow pulse — `box-shadow: 0 0 15px rgba(56,189,248,0.5), 0 0 30px rgba(56,189,248,0.3)` cycling via `animate-pulse`
- Active state (clicking while enabled): `active:scale-95` transition 150ms
- On click: button enters loading state immediately (optimistic), mana deducted via API, Pollinations.ai image fetched
- Success: image enters via `initial={{ opacity: 0, scale: 0.95 }} animate={{ opacity: 1, scale: 1 }}` — duration 400ms ease-out
- Button becomes "VISUAL GUIDANCE ACQUIRED" badge: `bg-cyan-900/30 border border-cyan-500/30 text-cyan-400 text-[10px] font-mono uppercase tracking-[0.3em]`

### Insufficient mana (shake)
- Shake animation: `keyframes: { 0%: translateX(0), 20%: translateX(-4px), 40%: translateX(4px), 60%: translateX(-4px), 80%: translateX(4px), 100%: translateX(0) }` — 300ms
- Implemented via Framer Motion `animate` with conditional trigger on click
- Tooltip: absolute positioned above button — `text-[10px] font-mono text-red-400 bg-black/80 px-2 py-1 rounded border border-red-500/30`

### Close modal
- X button top-right: `text-muted-foreground hover:text-white transition-colors p-2` — matches WorkoutEngine close button pattern exactly
- Modal dismisses via `AnimatePresence` exit animation — no confirmation needed

|---------|------|
| Guide button label (tooltip) | "View Exercise Guide" |

| Guide loading state | "THE SYSTEM IS ACCESSING COMBAT DATA..." |
| Steps section header | "STEPS" |
| Mistakes section header | "MISTAKES" |
| Breathing section header | "BREATHING" |
| System tip section header | "THE SYSTEM" |
| Mistake prefix | "HUNTER WARNING:" |

| Visual unlock button (loading) | "THE SYSTEM IS GENERATING VISUALS..." |
| Visual unlock button (complete) | "VISUAL GUIDANCE ACQUIRED" |

Voice rules:
- All caps section headers: Orbitron / Share Tech Mono uppercase
- THE SYSTEM speaks in second-person: "THE SYSTEM" as subject, not "I" or "We"
- HUNTER WARNING voice: imperative, terse, like a system alert
- Closing tip: motivational, boss flavor, present tense assertion (e.g., "A true hunter masters form before chasing reps.")

Source: 15-CONTEXT.md specifics section, Phase 13 established voice.

## Animation Contracts

| Interaction | Variant | Duration | Easing |
|-------------|---------|----------|--------|
| Modal open | opacity 0→1, scale 0.95→1 | 200ms | cubic-bezier(0.4,0,0.2,1) |
| Modal close | opacity 1→0, scale 1→0.95 | 150ms | cubic-bezier(0.4,0,0.2,1) |
| Image reveal (visual unlock) | opacity 0→1, scale 0.95→1 | 400ms | ease-out |
| Modal expand (image insertion) | height auto, smooth | 300ms | ease-in-out (layout animation) |
| Mana shake | translateX ±4px × 3 | 300ms | linear |
| Glow pulse (unlock button hover) | box-shadow oscillation | 2s | ease-in-out, infinite |
| Loading skeleton | opacity 0.3→0.7 | 1.5s | ease-in-out, infinite alternate |

All animations use Framer Motion — consistent with WorkoutEngine, BossEvent, RankUpCeremony patterns.

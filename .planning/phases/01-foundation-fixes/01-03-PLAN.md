## Plan: 01-03 — UserService Security + Starter Items + Intensity Rank Fix

### Context

**BUG-02**: `src/lib/services/userService.ts` imports from `@/lib/supabase` (anon key client) and calls `supabase.from("users").insert(...)` and `supabase.from("user_stats").upsert(...)` directly. If Supabase RLS policies enforce `auth.uid() = id`, these writes can fail in race conditions post-signup. The correct approach is to POST to the `/api/user` server route (which uses service role key).

**BUG-04**: `createUser()` in `userService.ts` never calls `seedStarterItems()` or `grantStarterItemsToUser()` from `inventoryService.ts`. New hunters have empty STORAGE tab.

**BUG-09**: In `src/lib/gameReducer.ts`, the `calculateIntensityRank` function has `if (ratio >= 4.0) return "C"` — since `ratio = flawlessReps/reps` is always 0–1, this condition is never true. The correct threshold is `>= 0.40`.

### Files to Modify
- `src/lib/services/userService.ts`
- `src/lib/gameReducer.ts`

### Tasks

#### Task 1: Fix calculateIntensityRank in gameReducer.ts
In `src/lib/gameReducer.ts`, find the `calculateIntensityRank` function at the bottom of the file:
- Change `if (ratio >= 4.0) return "C"` → `if (ratio >= 0.40) return "C"`
- This single character fix restores correct C-rank calculation

#### Task 2: Route createUser through /api/user server route
In `src/lib/services/userService.ts`:
- Replace the direct Supabase insert in `createUser` with a `fetch("/api/user", { method: "POST", body: JSON.stringify({...}) })` call
- Keep the `loadUser` function as-is (reads are fine with anon key + RLS)
- Remove the direct `supabase.from("users").insert(...)` and `supabase.from("user_stats").upsert(...)` calls from `createUser`
- The `/api/user` POST route already handles the upsert and stats seeding — use it

Note: The existing `/api/user` POST handler already does `upsert` + seeds `user_stats`. So after routing through it, `createUser` just needs to POST and check for success.

#### Task 3: Call starter item grant after user creation
In `src/lib/services/userService.ts`:
- Import `seedStarterItems`, `grantStarterItemsToUser` from `@/lib/services/inventoryService`
- After a successful user creation (after the /api/user POST returns 201), call:
  ```ts
  await seedStarterItems();              // idempotent: only seeds if items table is empty
  await grantStarterItemsToUser(params.id); // grants starter set to this specific user
  ```
- Wrap in try/catch — starter item failure should not block the createUser success return

#### Task 4: Verify flow in page.tsx
Confirm that `handleAwaken` in `src/app/page.tsx` still calls `createUser` correctly.
The interface hasn't changed, so no updates needed — just verify the import chain works.

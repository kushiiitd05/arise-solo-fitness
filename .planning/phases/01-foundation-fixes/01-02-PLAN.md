<objective>
Fix the critical bug where completing a daily quest awards XP but never updates the user's level or hunter_rank in the database. Currently /api/quests/complete only does current_xp += xpEarned with no level-up loop, so level resets to the DB value on every page refresh. Add the same level-up loop that exists in /api/xp/award to ensure the users table always reflects the true level and rank.
</objective>

## Plan: 01-02 — Quest Completion Level-Up Persistence

### Context
`src/app/api/quests/complete/route.ts` awards XP correctly but only runs:
```ts
await supabase.from("users").update({ current_xp: user.current_xp + xpEarned }).eq("id", userId);
```

This never processes level-ups. The `gameReducer` on the client handles level-up in memory but the DB stays stale. On refresh, the user's level reverts to whatever is in the database.

The fix is to inline the level-up loop (identical to what `/api/xp/award` does) and update `level`, `hunter_rank`, and `current_xp` in a single DB update.

### Files to Modify
- `src/app/api/quests/complete/route.ts`
- `src/lib/game/xpEngine.ts` (no changes needed — import from here)

### Tasks

#### Task 1: Import xpEngine functions in quest complete route
In `src/app/api/quests/complete/route.ts`:
- Add import: `import { xpForLevel, rankFromLevelAndXp } from "@/lib/game/xpEngine";`

#### Task 2: Add level-up loop to XP award section
Replace the existing XP update block (the `if (user)` section) with:
```ts
if (user) {
  let xp = user.current_xp + xpEarned;
  let level = user.level;
  let statPointsAwarded = 0;

  // Level-up loop
  while (xp >= xpForLevel(level)) {
    xp -= xpForLevel(level);
    level++;
    statPointsAwarded += 3;
  }

  const newRank = rankFromLevelAndXp(level, user.current_xp + xpEarned);

  await supabase
    .from("users")
    .update({ current_xp: xp, level, hunter_rank: newRank })
    .eq("id", userId);

  // Award stat points if leveled up
  if (statPointsAwarded > 0) {
    const { data: stats } = await supabase
      .from("user_stats")
      .select("available_stat_points")
      .eq("user_id", userId)
      .maybeSingle();
    if (stats) {
      await supabase
        .from("user_stats")
        .update({ available_stat_points: (stats.available_stat_points || 0) + statPointsAwarded })
        .eq("user_id", userId);
    }
  }
}
```

#### Task 3: Add leveledUp and newLevel to response
Include `leveledUp`, `newLevel`, `newRank`, `statPointsAwarded` in the JSON response so the client can dispatch the correct action.

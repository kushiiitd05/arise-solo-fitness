## Spacing Scale

All values are multiples of 4. These are the existing project tokens from `globals.css` and observed spacing in `ShadowArmy.tsx`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, rank badge padding, inline micro-spacing |
| sm | 8px | Icon-to-label gaps, chip padding, rank badge margin |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, card grid gap, header bottom margin |
| xl | 32px | Layout gaps between major panel zones |
| 2xl | 48px | Major section breaks, footer panel top margin |
| 3xl | 64px | Page-level spacing, hero areas |

Exceptions:
- ARISE button: `px-32px py-16px` (8×4 / 4×4 — fits scale)
- Shadow detail panel close button: 8px padding (fits scale)
- Rank badge: `px-8px py-4px` — implementation note: the `.rank-badge` utility class in `globals.css` must use `padding-top: 4px; padding-bottom: 4px` (Tailwind `py-1`), not `py-0.5` (2px)
- Stat chip header row: 16px horizontal gap between TOKENS and ARMY POWER values (fits scale)

Source: `src/app/globals.css` spacing tokens, `src/components/arise/ShadowArmy.tsx` observed padding values

## Component Inventory

All components are existing or extensions of existing components. No new top-level components are introduced.

### 1. ShadowArmy.tsx — Header Stat Chip (NEW element)

Renders `TOKENS: {n} | ARMY POWER: {n,nnn}` in the panel header, replacing (or extending) the current right-aligned ARISE button row.

Layout: `flex items-center gap-4` between TOKENS span and ARMY POWER span. A `|` divider in `text-white/20` between them.

```
[TOKENS: 2]  [|]  [ARMY POWER: 2,840]
```

Exact classes (from CONTEXT.md decision):
```tsx
<div className="flex items-center gap-4 text-[9px] font-system font-black tracking-[0.3em] uppercase">
  <span className="text-[#7C3AED]">TOKENS: {extractionTokens}</span>
  <span className="text-white/20">|</span>
  <span className="text-[#E2E8F0]">ARMY POWER: {armyPower.toLocaleString()}</span>
</div>
```

Placement: Same header row as the ARISE button, right-aligned, stacked below ARISE button on mobile.

Source: CONTEXT.md — "army power display" decision

### 2. ARISE Button — Token-Gated Disabled State

When `extractionTokens === 0`: `disabled={true}`, `opacity-30` already applied via existing `disabled:opacity-30` class. No shake animation — CONTEXT.md defers this to Claude's discretion. Recommendation: add a tooltip on the disabled button:

```tsx
title={extractionTokens === 0 ? "Defeat a boss to earn extraction tokens" : undefined}
```

This is a native HTML tooltip — zero additional code, zero animation overhead.

When `extractionTokens > 0` and extraction is in progress: existing `Loader2` spinner continues unchanged.

Source: CONTEXT.md — "ARISE button is disabled (opacity-30) when `extraction_tokens === 0`"

### 3. Extraction Overlay Message — Updated Copy

Replace the raw string `message` state with the structured notification via `ADD_NOTIFICATION`. The inline overlay message in the component may be removed or kept as a secondary visual. If kept, update copy to match:

- Success: `SHADOW EXTRACTED: {name}` (space-separated, not underscore)
- Failure: `EXTRACTION FAILED` (space-separated, not underscore)

The overlay uses existing styles unchanged. Copy change only.

Source: CONTEXT.md — "SHADOW EXTRACTED: Igris" / "EXTRACTION FAILED" (space-separated in CONTEXT.md)

### 4. Shadow Card — Existing (no visual change)

Shadow cards are complete and correct. No visual changes in Phase 10. The only behavioral change is that `persistentShadows` is refreshed post-extraction via `onExtractionChange` callback.

### 5. Empty State (no shadows extracted yet)

Existing empty state copy and visual are retained as-is. The clickable area already calls `handleArise`.

When `extractionTokens === 0` and army is empty: empty state remains visible but ARISE button is disabled. The empty state does NOT need a different copy for the zero-token condition — the button tooltip handles this.

## Accessibility

Per `design-system/arise/MASTER.md` pre-delivery checklist:

- All clickable elements must have `cursor-pointer` (existing `handleArise` and card `onClick` are covered)
- ARISE button disabled state: `disabled` attribute is set — browser handles focus/aria correctly
- Hover transitions: 150–300ms minimum (existing framer-motion transitions comply)
- Focus states: existing `border` transitions provide visible focus — ensure `focus-visible` ring on ARISE button: `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-[#7C3AED]/60`
- `prefers-reduced-motion`: existing framer-motion AnimatePresence should respect this via CSS `@media (prefers-reduced-motion: reduce)` — add if not present
- No emoji as icons: existing code uses emoji as decorative shadow images (`shadow.image` — emoji). This is acceptable as decorative content, not icons. All functional icons (Ghost, Loader2, PlusSquare, X, Zap, Shield, Sword) are Lucide SVGs — correct
- Minimum text contrast: `#7C3AED` on `#030308` = 4.67:1 (passes AA). `#E2E8F0` on `#030308` = 14.6:1 (passes AAA)

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Source: `design-system/arise/MASTER.md` spacing tokens — confirmed consistent with component usage in `RankUpCeremony.tsx` (p-8 = 32px, mb-12 = 48px, gap-8 = 32px).

Exceptions:
- Touch targets for dismiss button and chapter list items: minimum 44px height. Chapter list items use `p-5` (20px) which yields ~40px height — add `min-h-[44px]` to chapter item rows and ceremony dismiss button. Source: `RankUpCeremony.tsx` already enforces `min-h-[44px]` on its dismiss button.
- Ceremony component internal spacing: `mb-12` (48px) above badge reveal, `mb-16` (64px) below badges, `mb-10` (40px) below reward card — matches RankUpCeremony reference exactly.

## Component Inventory

### New components this phase

#### ChapterUnlockCeremony

Full-screen overlay rendered in `page.tsx` when a chapter unlock fires (parallel to RankUpCeremony render pattern).

**Structure:**
```
fixed inset-0 z-[200]  (same z-index tier as RankUpCeremony)
├── Background: #030308 with radial-gradient ellipse (cyan tint, NOT gold)
│   style={{ background: `radial-gradient(ellipse at center, rgba(6,182,212,0.08), #030308 70%)` }}
├── Header (motion, delay 0.2s):
│   └── Orbitron 28px font-black tracking-[0.25em] uppercase text-[#06B6D4]
│       "CHAPTER UNLOCKED"
├── Chapter reveal block (motion, delay 0.5s):
│   └── system-panel, p-8, border-[#06B6D4]/40, max-w-sm, text-center
│       ├── Chapter number: font-mono text-[10px] text-[#94A3B8] tracking-widest uppercase "CHAPTER {N}"
│       ├── Chapter title: Orbitron text-[20px] font-black text-[#E2E8F0] uppercase mt-2
│       └── Rarity badge (if applicable): font-mono text-[10px] uppercase tracking-widest (color from chapter.rarity)
├── External link status (motion, delay 0.9s):
│   ├── IF valid URL: font-mono text-[11px] text-[#06B6D4] — "SOURCE AVAILABLE — TAP TO READ"
│   └── IF no URL: font-mono text-[11px] text-[#94A3B8] — "SOURCE NOT YET AVAILABLE"
└── Dismiss button (motion, delay 1.2s):
    └── min-h-[44px] px-8 border font-orbitron text-[12px] font-black tracking-[0.25em] uppercase
        borderColor: #06B6D4, color: #06B6D4
        hover: bg-[#06B6D4]/10
        label: "ACKNOWLEDGE CHAPTER"
```

**Props interface:**
```tsx
interface ChapterUnlockCeremonyProps {
  chapterTitle: string;
  chapterNumber: number;
  externalUrl: string | null;
  onDismiss: () => void;
  dispatch: React.Dispatch<any>;
}
```

**Animation timing (mirrors RankUpCeremony):**
- Overlay fade-in: `duration: 0.4`
- Header slide down: `initial: { opacity: 0, y: -20 }`, `delay: 0.2`
- Chapter reveal: `type: "spring", stiffness: 200, damping: 15, delay: 0.5`
- Link status fade: `duration: 0.4, delay: 0.9`
- Dismiss button fade: `duration: 0.3, delay: 1.2`

Source: `RankUpCeremony.tsx` — structural template, swap gold for cyan, replace rank badges with chapter card.

#### Dashboard chapter list — onClick upgrade

Existing chapter list items in Dashboard (`chapters?.slice(0, 5).map(...)`) require an `onClick` handler added to unlocked rows.

**Interaction contract:**
- Unlocked row with valid `externalUrl`: `onClick` calls `window.open(ch.externalUrl, '_blank')`. Row already has `cursor-pointer`.
- Unlocked row with null/invalid URL: `onClick` dispatches `ADD_NOTIFICATION` with type `INFO`, title `"SOURCE NOT YET AVAILABLE"`, body `"This chapter has been unlocked but the source URL is not confirmed. Check back soon."`. No external navigation.
- Locked row: no `onClick`. Already has `grayscale opacity-20` styling. No hover affordance.

**Visual additions to unlocked rows:**
- Add an external-link icon (use `ExternalLink` from lucide-react, size 14) next to the existing `Sparkles` icon when `ch.externalUrl` is a valid URL. Color: `text-[#06B6D4]`.
- When URL is null: replace Sparkles with a lock-open icon (`Unlock` from lucide-react, size 16, color `text-[#94A3B8]`) + no ExternalLink icon.

Source: `Dashboard.tsx` line 334, `12-CONTEXT.md` decisions (external link only, "Source not yet available" fallback).

## Interaction States

### Chapter list item states

| State | Visual |
|-------|--------|
| Locked | `grayscale opacity-20 border-white/5` — no cursor, no hover affordance |
| Unlocked, valid URL | `border-white/10 bg-white/[0.03] hover:bg-white/[0.06] cursor-pointer` + Sparkles icon (cyan) + ExternalLink icon (cyan) |
| Unlocked, no URL | `border-white/10 bg-white/[0.03] hover:bg-white/[0.06] cursor-pointer` + Unlock icon (muted gray) — click fires INFO notification |
| Hover (unlocked) | Already handled by `hover:bg-white/[0.06]` — no additional glow needed |

### CHAPTER notification appearance

- Left-border: `#7C3AED` (purple) — inherits from CHAPTER/SHADOW branch in `SystemNotification.tsx`
- Icon: Ghost (lucide-react) — inherits from `isShadow` branch (CHAPTER treated same as SHADOW in current code)
- Dismiss duration: 6500ms (updated from 5000ms)
- Progress bar color: `#7C3AED`

### ChapterUnlockCeremony trigger

Ceremony is triggered when a CHAPTER unlock event fires server-side. The response from POST /api/quests/update or POST /api/boss/complete includes the newly unlocked chapter data. `page.tsx` sets `showChapterUnlock: true` and `chapterUnlockData: { chapterTitle, chapterNumber, externalUrl }`. Ceremony renders over everything at `z-[200]`. On dismiss: `setShowChapterUnlock(false)`, ceremony unmounts.

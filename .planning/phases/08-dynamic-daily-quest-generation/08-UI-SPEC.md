## Component Inventory — Phase 8 Additions

These are the net-new visual elements Phase 8 adds to `QuestBoard.tsx`. Existing elements are not re-specified.

Primary focal point: the quest card list is the dominant visual anchor of the daily tab. The difficulty badge is a secondary orientation cue positioned top-right of each card; it must not compete with the quest name or XP reward for visual weight.

### 1. Difficulty Badge

Position: absolute top-right corner of each quest card, inside the card's `relative` wrapper.

```
.difficulty-badge
  position: absolute top-2 right-2   (8px from card edge)
  layout: inline-flex items-center gap-1
  padding: 2px 6px (px-1.5 py-0.5)
  border: 1px solid {color}/40
  background: {color}/10
  clip-path: same corner-cut as rank-badge in globals.css
  font: text-[9px] font-title font-black tracking-[0.15em] uppercase  (Micro role — Orbitron variant)
  text: EASY | NORMAL | HARD
  z-index: 20 (above card glow layer at z-10)
```

The badge must not overlap the XP reward display (top-right of the card content row). The card content row sits inside `relative z-10 flex items-center gap-6` — the badge is positioned relative to the card outer div. Card currently has no top-right absolute element, so placement is clear.

### 2. Quest Count Header Update

The ACTIVE_GOALS counter currently shows `N/M Verified Goals`. No change to layout — the completed/total counts naturally update when quests change. No new element needed.

### 3. Daily Quest Loading State (no change)

Existing `SYNCHRONIZING_WITH_SYSTEM_SERVER...` loading state already handles the POST /api/quests/daily fetch. No visual change.

## Interaction Contracts

| Trigger | Behaviour |
|---------|-----------|
| QuestBoard mounts, daily tab active | POST /api/quests/daily fires (existing); returned quest objects now include `difficulty` field |
| Quest card renders | If `quest.difficulty` exists, render badge in top-right corner; if missing (fallback), render no badge |
| Quest completed | Badge stays visible, card opacity drops to 0.6 (existing pattern) — badge color does not change on completion |
| Tab switch to story/emergency | No quest generation interaction — these tabs are unchanged stubs |

Motion: no new framer-motion animations. Difficulty badge renders statically. Badge does not animate in separately from the card.

## Accessibility

- Difficulty badge must include `aria-label="Difficulty: EASY"` (or NORMAL/HARD). The colour alone is not sufficient for colourblind users.
- Badge must not be the only indicator of difficulty — the label text `EASY/NORMAL/HARD` satisfies this.
- Existing quest card keyboard focus is inherited from outer `div` — no interactive change, no new tabIndex required.
